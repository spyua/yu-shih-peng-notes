Context Window 深解

理解 AI 的「工作記憶」如何被消耗，以及滿了之後會發生什麼事。

📖 你會學到什麼
  • Context Window 是什麼，以及它對 AI 的限制意義
  • Context 由哪三層組成，以及哪一層才是真正的大頭
  • Context 接近上限時會出現哪四種問題，以及它們如何悄悄影響品質
  • 主動管理 Context 的核心指令（/clear、/compact）與背後邏輯

什麼是 Context Window？

Context Window 是 LLM 單次處理能夠「看到」的文字總量上限。你可以把它想成 AI 的工作記憶——就像人類在解題時，腦中能同時記住的資訊量是有限的。

生活比喻：書桌的面積

想像你在一張書桌上解題。Context Window 就是書桌的大小——你只能把「放得上桌面」的資料納入考量。超出桌面的書本，你就算知道它存在，也沒辦法同時參考。

各模型的 Context Window 大小

模型系列Context Window約等於
GPT-5 系列 128K – 1M tokens 約 10 萬 – 75 萬中文字
Claude 4 系列 200K tokens 約 15 萬中文字

看起來很多？實際上在 Agent Tool 場景中，Context 會被快速消耗——而且消耗的主力，並不是你打的對話。

Context 的三層結構

在 Agent Tool 的使用場景中，Context 不只是你打的字。它由三層內容組成，而最大的消耗源往往是第三層。

第三層是 Context 被快速消耗的主因。 每當 Agent 讀取一個 500 行的程式碼檔案，就是幾千個 tokens 的消耗。如果一個任務需要讀取 10 個檔案，Context 很快就會接近上限——而許多人對此毫無察覺。

Context 滿了會怎樣？

當 Context 接近上限時，問題不會突然爆發，而是悄悄地讓 AI 品質下降。這是最危險的情況——你以為 AI 在正常運作，但它已經開始「忘記」重要資訊。

🧠
品質下降
AI 開始「忘記」早期對話中的重要資訊，回答變得不一致，甚至與之前的決定矛盾。
✂️
截斷風險
部分 Agent Tool 會自動截斷舊的對話，可能丟失關鍵指令或前置架構決策。
🗜️
自動壓縮
部分 Agent Tool 會自動 compact，但壓縮品質不可控，可能遺失細節。
🚫
直接報錯
到達硬上限時，Session 無法繼續，必須強制開啟新的對話。
主動管理才是正解

Context Management 是 Agent Tool 操作的核心能力。/clear 可以清空 Context、釋放工作記憶；/compact 可以在不清空的前提下壓縮舊的對話。在問題發生之前主動介入，遠比事後補救有效。

操作指令對照

操作背後的原因

/compact 壓縮 Context，在不清空的前提下釋放空間
切換 Model 不同任務需要不同的推理深度與速度

Agent 讀檔要有策略 每讀一個檔案都在消耗寶貴的 Context 空間
❓ 問答練習
Context Window 和「AI 記憶力」是同一件事嗎？ +
不完全是。Context Window 是單次對話中 AI 能「看到」的文字總量上限，它更像是「工作桌的面積」而非「長期記憶」。AI 在 Context 之外沒有任何記憶——每次對話都是全新開始，它「記得」的一切都來自當前 Context 裡的內容。記憶力的比喻容易讓人誤以為 AI 能累積經驗，但實際上它只能參考「桌上現有的資料」。
為什麼第三層（工具回傳）是 Context 的最大消耗源？ +
因為 Agent 讀取檔案時，是把整個檔案內容塞進 Context，而不是摘要。一個 500 行的程式碼約 3,000–5,000 tokens，grep 搜尋結果可能幾百行，指令的 stdout/stderr 輸出可能更長。相較之下，你打的對話通常只有幾百 tokens，System Prompt 頂多幾千。一個複雜任務下來，工具回傳佔 Context 的 50–80% 是很常見的。
/clear 和 /compact 的使用時機有什麼差別？ +
/clear 是完全清空 Context，重新開始——適合當前任務已完成、要切換到全新任務時使用。/compact 是壓縮並保留對話摘要——適合同一個任務還沒結束、但 Context 已經快滿的情況，它能釋放空間同時保留關鍵脈絡。選擇的邏輯很簡單：任務還沒完成用 compact，已完成要換任務用 clear。
Context 快滿時，AI 表現變差的跡象有哪些？ +
常見的跡象包括：前後不一致（與前幾輪說的不同）、重複定義（重新宣告已存在的變數或函式）、忽略前置條件（忘記你在早期對話中設定的限制或需求）、回答變得籠統（不再引用具體的程式碼細節，改用模糊描述）。這些都是 AI 開始「忘記」早期 Context 的信號，應該立即 /compact 或拆分任務。
